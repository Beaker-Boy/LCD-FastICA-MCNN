# 掘进机主轴承振动信号故障诊断

## 项目概述
这是一个基于振动信号的故障诊断系统。它通过一个 PyQt5 图形用户界面（GUI）接收用户的操作指令，依次执行 LCD-FastICA 信号处理、数据张量构建和 MCNN 模型训练，最终生成一个能够根据振动信号判断设备故障类型的深度学习模型。

## 技术栈

1. **GUI框架**：PyQt5
2. **核心算法库**：Python, PyTorch, scikit-learn, NumPy, SciPy
3. **数据读写库**：nptdms (用于读取 TDMS), SciPy (用于读写 MAT)
4. **数据存储格式**:
   - **原始数据**: `.tdms`
   - **中间特征**: `.mat`
   - **模型输入**: `.pt` (PyTorch Tensor)

## 数据模型
由于项目不涉及传统关系型数据库，本系统的数据模型主要体现在多维数组（Tensor/Array）的结构与流转：
1. **原始数据 (Raw Data)**：从 `.tdms` 文件中读取指定通道的一维振动信号时间序列。
2. **中间特征 (Intermediate Features)**：经过 LCD-FastICA 处理后，生成的独立分量被保存为 `.mat` 文件。每个文件包含一个形状为 `(signal_length, num_components)` 的数组。
3. **模型输入数据 (Input Tensors)**：将多个 `.mat` 文件中的特征组合、切片、增广成多个样本，形成形状为 `(N_samples, N_channels, sample_length)` 的张量，并保存为 `.pt` 文件供模型直接读取。
4. **标签数据 (Labels)**：根据用户在 GUI 中为每个 `.tdms` 文件选择的标签（如 'A', 'B' 等）生成。在数据构建阶段，为该文件产生的所有样本赋予相同的标签，并最终保存为 `.pt` 文件。

## 核心功能模块
根据项目架构，核心模块分为以下四个部分：
1. **GUI 交互模块 (`main_window.py`)**：
   - 提供图形界面，引导用户选择原始 `.tdms` 文件、指定通道、选择标签和训练模式。
   - 作为总控制器，按顺序调用信号处理、张量构建和模型训练三大核心流程。
2. **数据读取与张量构建模块 (`张量构建.py`)**：
   - 读取 `LCD-FastICA` 模块输出的多个 `.mat` 特征文件。
   - 将多组特征融合成一个多通道长张量。
   - 对长张量进行滑窗切片，生成大量带有重叠的训练样本。
   - 按 80/20 的比例划分训练集和验证集，并与标签一同保存为 `.pt` 文件。
2. **信号处理与降噪模块 (LCD-FastICA)**：
   - 对输入的原始信号进行低通滤波。
   - **LCD (局部特征尺度分解)**：将非平稳、非线性的复杂振动信号分解为若干个内禀尺度分量 (ISC)。
   - **FastICA (快速独立成分分析)**：对分解出的 ISC 分量进行盲源分离，提取出更具代表性的独立特征成分，并保存为 `.mat` 文件。
3. **深度学习模型构建模块 (MCNN) (`MCNN.py`)**：
   - 构建一个多尺度自适应卷积神经网络 (MSASCnn)。
   - **宽卷积层 (WideConvLayer)**：使用 128 和 64 两种不同大小的卷积核并行提取特征。
   - **多尺度自适应卷积块 (MSASCblock)**：通过三个并行的卷积层进一步提取多尺度信息。
4. **模型训练与评估模块 (`模型训练.py`)**：
   - 加载 `.pt` 格式的训练和验证数据集。
   - 定义适合多分类任务的损失函数（如 CrossEntropy）及优化算法。
   - 支持从零开始训练或加载已有模型继续训练。
   - 实现标准的训练-验证循环，并在每个 epoch 结束后打印 Loss。

## 实施路线图
*   **阶段 1：数据探索与基建**。完成 `.npy` 数据集的加载脚本，验证数据维度的正确性，并跑通基础的预处理流程。
*   **阶段 2：信号处理算法验证**。编写并调试 LCD 与 FastICA 算法，对单一故障样本进行处理，绘制处理前后的波形图/频谱图，验证去噪与特征提取效果。
*   **阶段 3：流水线整合与特征生成**。将整个数据集送入 `LCD-FastICA` 流水线进行批量处理，并将处理后的特征保存为新的 `.npy` 文件，以加速后续神经网络的训练。
*   **阶段 4：MCNN 网络搭建与初步训练**。基于 PyTorch 或 TensorFlow 搭建多尺度 CNN 网络，接入处理好的特征数据，打通前向传播与反向传播的训练链路。
*   **阶段 5：参数调优与可视化评估**。调整网络超参数（如卷积核大小、层数、学习率等），在测试集上验证准确率，并输出混淆矩阵等可视化报告。

## 现有代码(v1.0)与设计的差异及重构计划
根据对 `一些实现方法` 目录下初始脚本的审查，发现当前实现与设计意图存在多处严重偏差，且包含一些致命错误，导致整个流程无法正常工作。本节记录了这些问题，并作为后续重构的参考依据。

### 1. 致命问题：流程中断与逻辑错误
*   **路径硬编码与错乱 (P0 - 阻塞性)**:
    *   **问题**: `LCD_FastICA.py` 将输出保存到 `数据\` 目录，而 `main_window.py` 却让后续步骤从 `处理后数据\` 目录读取。路径不匹配导致流程中断。
    *   **重构目标**: 废除所有硬编码路径。流程中所有文件的输入/输出路径都应由 `main_window.py` 统一管理和传递。

*   **标签逻辑完全错误 (P0 - 阻塞性)**:
    *   **问题**: `张量构建.py` 将单个标签赋予了数据集中所有样本，导致无法训练分类器。
    *   **重构目标**: 重新设计数据处理流程，确保能为来自不同源文件的样本正确赋予其各自的标签。需要一套机制来管理多类别数据的累积和构建。

*   **外部项目依赖 (P0 - 阻塞性)**:
    *   **问题**: `模型训练.py` 通过绝对路径 `sys.path.append` 依赖了另一个不相关的项目 (`TBM-MainBearing`) 来导入模型。
    *   **重构目标**: 移除该依赖，改为从本地的 `MCNN.py` 文件中正确导入模型。

### 2. 架构与逻辑问题
*   **工作流矛盾 (P1 - 主要)**:
    *   **问题**: GUI设计为一次处理一个文件，而数据构建脚本却期望一次性处理一个文件夹下的所有类别数据。这个矛盾使得多分类数据集的构建流程不明确。
    *   **重构目标**: 明确数据集构建的工作流。例如，GUI可以增加“添加文件到处理列表”的功能，最后统一处理列表中的所有文件，以构建一个包含多类别的数据集。

*   **算法细节疑问 (P2 - 次要)**:
    *   **问题**: FastICA处理时，仅使用了第一个ISC分量和原始信号，这与文档描述不完全相符，其合理性需要验证。
    *   **重构目标**: 澄清此算法逻辑。如果该方法有效，需在文档中补充说明；如果无效，则应修正为对多个关键ISC分量进行分析。

### 3. 代码质量问题
*   **文件保存Bug (P1 - 主要)**:
    *   **问题**: `LCD_FastICA.py` 将相同结果重复写入多个文件。
    *   **重构目标**: 修正此Bug。输出文件名应根据输入文件名动态生成，确保唯一性。

*   **操作冗余 (P2 - 次要)**:
    *   **问题**: 数据归一化在 `张量构建.py` 和 `模型训练.py` 中被执行了两次。
    *   **重构目标**: 移除其中一次归一化操作，统一在数据加载后、训练前进行。

*   **缺少关键评估指标 (P1 - 主要)**:
    *   **问题**: `模型训练.py` 只输出了Loss，无法有效评估模型性能。
    *   **重构目标**: 在验证循环中增加计算并输出**准确率(Accuracy)**。长远来看，应实现如精确率、召回率、混淆矩阵等更全面的评估手段。
